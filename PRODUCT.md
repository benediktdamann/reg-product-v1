# PRODUCT

## Purpose

Provide a lightweight foundation for a regulatory-oriented product MVP.

## Current user-facing scope (implemented)

- View a minimal landing page.
- View a static "Sources" table at `/sources`.

## Out of scope or not yet confirmed

- User authentication and authorization.
- Data ingestion pipelines.
- Persisted source management or CRUD workflows.
- Search, filtering, or ranking functionality.
- Production-grade monitoring and observability.

## Current constraints

- Frontend data is static and in code.
- Backend platform (`supabase/`) is currently a placeholder.
- Worker orchestration exists, but product integration is not yet confirmed.

## Success criteria for this phase

- Establish operating structure and execution consistency.
- Keep implementation tasks small and traceable.
- Improve delivery confidence through minimal QA and work-order standards.
